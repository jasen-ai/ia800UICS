# UICS 媒体生成使用数据说明

本文档说明**图片生成**、**视频生成**、**音频生成**的 **API 输入参数**、**ComfyUI 时涉及的参数**，以及 **Excel 数据来源**（表与列），便于对接接口与排查问题。

- **API**：请求体为 JSON，字段名与下表一致；需登录后在请求头中携带 `X-Session-ID`。
- **ComfyUI**：当 `generator_type=comfyui` 时，服务端会向 ComfyUI 传递或使用表中列出的参数（部分来自 API，部分来自 Excel 或本地文件）。

---

## 一、图片生成

### 1.1 API 输入参数（请求体）

对应接口：**POST /api/generate/image**（请求头需带 `X-Session-ID`）。

| 参数 | 类型 | 必填 | 说明 | 默认 |
|------|------|------|------|------|
| output_dir | string | 否 | 输出目录（服务端实际使用 OUTPUT_FOLDER，本参数仅作记录） | "./output" |
| episode_filter | string | 否 | 只处理指定剧集，如 EP01 | null |
| shot_filter | string | 否 | 只处理指定分镜，多个用逗号/空格分隔，如 EP01_SQ01,EP01_SQ02 | null |
| generator_type | string | 否 | 生成器：**comfyui** / **nanobanana** | "comfyui" |
| comfyui_server | string | 否 | ComfyUI 服务地址，仅当 generator_type=comfyui 时生效 | "127.0.0.1:8188" |
| generate_reference | bool | 否 | 是否生成参考图（图像提示词） | true |
| generate_first_frame | bool | 否 | 是否生成首帧图 | false |
| generate_last_frame | bool | 否 | 是否生成末帧图 | false |
| enable_prompt_expansion | bool | 否 | 是否启用提示词扩展（按角色附加特征） | true |

### 1.2 使用 ComfyUI 时涉及的参数（generator_type=comfyui）

UICS 调用 ComfyUI 时，会用到以下输入（来源：API 传参 / Excel / 本地目录）：

| 参数 | 来源 | 说明 |
|------|------|------|
| server_address | API：comfyui_server | ComfyUI 地址，如 127.0.0.1:8188 |
| prompt | Excel：图像汇总.图像提示词（或首帧/末帧提示词） | 正提示词；若启用扩展则追加角色汇总.图像提示词/视觉特征 |
| style_prefix | 代码默认或上游传入 | 风格前缀（如 "Pixel art style,"），文生图时拼在 prompt 前 |
| filename_prefix | 按分镜生成：{分镜号}_ref / _first / _last | 输出文件名前缀 |
| seed, steps, cfg | 可选，代码默认 | 采样种子、步数、CFG；写入工作流 KSampler |
| width, height | 可选，代码默认 | 分辨率；写入工作流 EmptySD3LatentImage 等 |
| reference_images | Excel + 本地目录解析 | 1～3 张：参考图1=场景id 图，参考图2=角色id 图，参考图3=图像汇总.参考图；有 1～3 张时使用 act_02_qwen_Image_edit-aigc-3-api.json，否则使用 z_image_workflow.json |

**工作流选择**：无参考图或参考图不是 1～3 张 → `z_image_workflow.json`（文生图）；参考图 1～3 张 → `act_02_qwen_Image_edit-aigc-3-api.json`（图生图）。

### 1.3 使用 Nano Banana 时涉及的参数（generator_type=nanobanana）

| 参数 | 来源 | 说明 |
|------|------|------|
| prompt | Excel：图像汇总.图像提示词 | 与 ComfyUI 相同逻辑，可扩展 |
| reference_images | Excel：图像汇总.参考图 等 | 1～3 张时为图生图，否则文生图 |
| filename_prefix | {分镜号}_ref / _first / _last | 输出文件名前缀 |
| 其他 | 配置/默认 | 依赖 nanobanana 的 API 与配置 |

### 1.4 数据来源：Excel 表与列

**主表：图像汇总**（工作表名：`图像汇总`，兼容旧名 `图像提示词`）

| 列名 | 用途 | 必填/说明 |
|------|------|-----------|
| 剧集id | 剧集过滤、输出归属 | 必填 |
| 分镜号 | 分镜唯一标识，用于过滤与输出文件名 | 必填，如 EP01_SQ01 |
| 场景名 | 用于解析参考图1：在场景汇总中查场景id，在场景图目录下找 {场景id}.png | 可选 |
| 角色 | 用于解析参考图2：在角色汇总中查角色id，在角色图目录下找 {角色id}.png；提示词扩展时从角色汇总取「图像提示词/视觉特征」 | 可选 |
| 场景内容 | 剧本场景描述（不参与生成逻辑，可做备注） | 可选 |
| 镜头类型 | 备注用 | 可选 |
| 图像提示词 | 生成参考图/首帧/末帧时使用的正提示词 | 生成参考图时必填 |
| 首帧提示词 | 仅当勾选「生成首帧」时使用 | 可选 |
| 末帧提示词 | 仅当勾选「生成末帧」时使用 | 可选 |
| 视频提示词 | 不参与图片生成，供视频生成使用 | 可选 |
| 备注 | 备注 | 可选 |
| 参考图 | 参考图3：逗号分隔的路径或文件名（相对路径以 output 为基准），1～3 张与场景/角色图一起参与图生图 | 可选 |

**关联表（用于参考图与提示词扩展）**

- **场景汇总**：列「场景名」「场景id」—— 图像汇总.场景名 匹配 场景汇总.场景名，得到 场景id，再在场景图目录下找 `{场景id}.png`（或 .jpg）。
- **角色汇总**：列「角色名」「角色id」「图像提示词」「视觉特征」「参考音色」等 —— 图像汇总.角色 匹配 角色汇总.角色名，得到 角色id 与角色特征；参考图2 在角色图目录下找 `{角色id}.png`；提示词扩展时把该角色的「图像提示词」或「视觉特征」追加到正提示词后。

### 1.5 输出文件命名

- 参考图：`{分镜号}_ref_*.png`（或 .jpg 等）
- 首帧：`{分镜号}_first_*.png`
- 末帧：`{分镜号}_last_*.png`

## 二、视频生成

### 2.1 API 输入参数（请求体）

对应接口：**POST /api/generate/video**（请求头需带 `X-Session-ID`）。

| 参数 | 类型 | 必填 | 说明 | 默认 |
|------|------|------|------|------|
| output_dir | string | 否 | 输出目录（服务端实际使用 OUTPUT_FOLDER） | "./output" |
| episode_filter | string | 否 | 只处理指定剧集 | null |
| shot_filter | string | 否 | 只处理指定分镜，多个逗号/空格分隔 | null |
| generator_type | string | 否 | 生成器：**comfyui** / **sora** | "comfyui" |
| comfyui_server | string | 否 | ComfyUI 服务地址，仅当 generator_type=comfyui 时生效 | "127.0.0.1:8188" |
| enable_prompt_expansion | bool | 否 | 是否扩展视频提示词（按角色） | true |
| reference_image_dir | string | 否 | 图生视频首帧/末帧图片所在目录（本地）；不传则用 output_dir | null |
| reference_audio_dir | string | 否 | 数字人 s2v 时，分镜音频所在目录（下存 {分镜号}.wav/.mp3）；不传则不做 s2v | null |
| workflow_path | string | 否 | 自定义工作流 JSON 路径（一般可不传，使用默认） | null |

### 2.2 使用 ComfyUI 时涉及的参数（generator_type=comfyui）

| 参数 | 来源 | 说明 |
|------|------|------|
| server_address | API：comfyui_server | ComfyUI 地址 |
| workflow_path | API 或默认 | 默认：i2v 用 act_video_wan2_2_14B_i2v-aigc-api.json；有末帧用 i2vse；s2v 用 act_video_wan2_2_14B_s2v-aigc-api.json |
| prompt | Excel：图像汇总.视频提示词 | 正提示词；可扩展 |
| negative_prompt | 代码默认 | 负提示词，写入工作流 CLIPTextEncode |
| image_path | 本地：reference_image_dir 下 {分镜号}_ref.* | 首帧图；上传至 ComfyUI 后写入 LoadImage |
| end_image_path | 本地：同目录下 {分镜号}_last.* | 末帧图；有则用 i2vse 工作流 |
| audio_path | 本地：reference_audio_dir 下 {分镜号}.wav/.mp3 | 仅 s2v 数字人；上传后写入 LoadAudio |
| seed, steps, cfg | 可选 | 写入 KSampler / KSamplerAdvanced |
| width, height, length, fps | 可选 | 写入 WanImageToVideo / WanFirstLastFrameToVideo / WanSoundImageToVideo 等 |
| filename_prefix | 按分镜：{分镜号}_video | 输出视频文件名前缀 |

**工作流选择**：有 reference_audio_dir 且存在该分镜音频 + 首帧图 → s2v；否则有首帧+末帧图 → i2vse；否则 → i2v。

### 2.3 使用 Sora 时涉及的参数（generator_type=sora）

| 参数 | 来源 | 说明 |
|------|------|------|
| prompt | Excel：图像汇总.视频提示词 | 与 ComfyUI 相同 |
| image_file | 本地：{分镜号}_ref.* | 可选，首帧图 |
| model, aspect_ratio, duration, size 等 | 配置/默认 | 见 Sora API 与 sora_video_client |

### 2.4 数据来源：Excel 表与列

**主表：图像汇总**

| 列名 | 用途 | 必填/说明 |
|------|------|-----------|
| 剧集id | 剧集过滤 | 必填 |
| 分镜号 | 分镜标识；与本地文件名对应 | 必填 |
| 视频提示词 | 图生视频的正向提示词（可被提示词扩展） | 有视频提示词的分镜才会生成 |
| 场景内容、场景名、角色等 | 仅用于提示词扩展时从角色汇总取特征 | 可选 |

**视频生成：参考图目录说明**

- **参考图**指图生视频的**首帧图**（必选）和**末帧图**（可选），来自**本地目录中的文件**，**不是** Excel 中的列。
- **首帧/末帧所在目录**：
  - 默认：**UICS 的 output 目录**（即服务端配置的 `OUTPUT_FOLDER`，如 `UICS/output`）。
  - 或 API 传入 **reference_image_dir** 时，使用该目录（可为绝对路径或相对 UICS 的路径）。
- **文件名约定**（与 图像汇总.分镜号 一一对应）：
  - 首帧：`{分镜号}_ref.png` / `{分镜号}_ref_00018.png` 等（.png / .jpg / .jpeg / .webp）。
  - 末帧：`{分镜号}_last.png` / `{分镜号}_last_00001.png` 等。
- 这些文件通常由 **图片生成** 先产出到 output（或指定目录），再在 **视频生成** 时被读取。

**本地文件（非 Excel）**

- **首帧图**：在 reference_image_dir（或 output_dir）下查找 `{分镜号}_ref.*`（.png / .jpg / .jpeg / .webp），或 `{分镜号}_ref_*.*`。通常由「图片生成」先产出。
- **末帧图**（可选）：同目录下 `{分镜号}_last.*` 或 `{分镜号}_last_*.*`。若存在末帧图，ComfyUI 使用首末双图工作流（i2vse）。
- **数字人 s2v**：reference_audio_dir 下 `{分镜号}.wav` 或 `{分镜号}.mp3` 等，与同分镜的首帧图一起送入 s2v 工作流。

### 2.5 输出文件命名

- 图生视频：`{分镜号}_video.mp4` 或 `{分镜号}_video_*.mp4`

### 2.6 示例

**请求体示例（POST /api/generate/video）**

```json
{
  "output_dir": "./output",
  "episode_filter": "EP01",
  "shot_filter": "EP01_SQ01",
  "generator_type": "comfyui",
  "comfyui_server": "127.0.0.1:8188",
  "enable_prompt_expansion": true,
  "reference_image_dir": null,
  "reference_audio_dir": null
}
```

- 不传 `reference_image_dir` 时，首帧/末帧在 output 目录下查找；不传 `reference_audio_dir` 时不做数字人 s2v。

**图像汇总表示例（与视频生成相关的列）**

| 剧集id | 分镜号   | 视频提示词                                                                 |
|--------|----------|----------------------------------------------------------------------------|
| EP01   | EP01_SQ01 | 林婉在咖啡厅窗边看手机，镜头缓慢推进，自然光，电影质感                     |
| EP01   | EP01_SQ02 | 李哲将卡片递给林婉，两人对视，固定镜头，柔和室内光                         |

**参考图目录与目录内文件示例**

- **目录**：例如 UICS 部署下为 `UICS/output`（即 OUTPUT_FOLDER）。或通过 API 传 `reference_image_dir`，如 `"/data/uics_output"`。
- **目录内文件示例**（与 图像汇总 中的 分镜号 对应）：

```
UICS/output/
  EP01_SQ01_ref_00018.png    ← 首帧（图生视频用）
  EP01_SQ01_first_00001.png  ← 首帧图（图片生成产出，与 ref 通常一致或同源）
  EP01_SQ01_last_00001.png   ← 末帧（有则用 i2vse）
  EP01_SQ02_ref.png
  EP01_SQ02_video.mp4        ← 视频生成输出
```

- 首帧图（图生视频必选）：上述目录下存在 `EP01_SQ01_ref.png` 或 `EP01_SQ01_ref_00018.png` 等，才会用该分镜做图生视频。
- 末帧图（可选）：若同目录下还有 `EP01_SQ01_last.png`，则使用首末双图工作流（i2vse）。
- 数字人 s2v：若传 `reference_audio_dir` 且该目录下有 `EP01_SQ01.wav`（或 .mp3），则用首帧图 + 该音频走 s2v 工作流生成数字人视频。

**Excel 数据例子（视频生成只用到图像汇总中的分镜号与视频提示词）**

视频生成**不**从 Excel 读「参考图路径」；参考图由**本地目录 + 文件名约定**确定。Excel 中只需填好 分镜号 与 视频提示词；首帧/末帧文件需事先存在于参考图目录中（通常由图片生成得到）。

| 剧集id | 分镜号   | 视频提示词                                                                 |
|--------|----------|----------------------------------------------------------------------------|
| EP01   | EP01_SQ01 | 林婉在咖啡厅窗边看手机，镜头缓慢推进，自然光，电影质感                     |
| EP01   | EP01_SQ02 | 李哲将卡片递给林婉，两人对视，固定镜头，柔和室内光                         |

- 上表填好后，若 `UICS/output`（或 reference_image_dir）下已有 `EP01_SQ01_ref.png`、`EP01_SQ02_ref.png`，视频生成会按分镜逐条用对应 ref 图 + 视频提示词生成视频。

**输出示例**

- 生成成功后，输出目录下会出现 `EP01_SQ01_video.mp4`（或 `EP01_SQ01_video_00001_.mp4`，服务端会规范为 .mp4）。

## 三、音频生成

### 3.1 API 输入参数（请求体）

对应接口：**POST /api/generate/audio**（请求头需带 `X-Session-ID`）。数字人声为 **POST /api/generate/audio_digital**（参数与 audio 类似，generator_type 固定为 comfyui）。

| 参数 | 类型 | 必填 | 说明 | 默认 |
|------|------|------|------|------|
| output_dir | string | 否 | 输出目录（服务端实际使用 OUTPUT_FOLDER） | "./output" |
| episode_filter | string | 否 | 只处理指定剧集 | null |
| shot_filter | string | 否 | 只处理指定分镜，多个逗号/空格分隔 | null |
| generator_type | string | 否 | 生成器：**volcengine** / **comfyui** | "volcengine" |
| config_path | string | 否 | 火山引擎等配置文件路径，仅 volcengine 时使用 | null |
| comfyui_server | string | 否 | ComfyUI 服务地址，仅当 generator_type=comfyui 时生效 | "127.0.0.1:8188" |
| workflow_path | string | 否 | ComfyUI 工作流 JSON 路径；comfyui 时默认 Qwen3-TTSVoiceCloneAPI.json | null |

### 3.2 使用 ComfyUI 时涉及的参数（generator_type=comfyui，含数字人声）

| 参数 | 来源 | 说明 |
|------|------|------|
| server_address | API：comfyui_server | ComfyUI 地址 |
| workflow_path | API 或默认 | 默认 Qwen3-TTSVoiceCloneAPI.json |
| ref_audio_path | Excel：角色汇总.参考音色（或参考图）；或音频汇总.参考样本；或角色名作路径 | 参考音频文件路径，上传至 ComfyUI 后写入 LoadAudio |
| target_text | Excel：音频汇总.音频内容 | 要合成的文本 |
| filename_prefix | Excel：音频汇总.音频id | 输出文件名前缀 |
| target_language | 代码默认 | 如 "Chinese" |
| temperature, top_p, top_k, repetition_penalty, max_new_tokens | 代码默认 | Qwen3TTSVoiceClone 节点参数 |
| output_mode | 代码默认 | 如 "Concatenate (Merge)" |
| seed, ref_text, instruct | 可选 | 工作流中可配置 |

### 3.3 使用火山引擎时涉及的参数（generator_type=volcengine）

| 参数 | 来源 | 说明 |
|------|------|------|
| text | Excel：音频汇总.音频内容 | 要合成的文本 |
| voice_type | Excel：角色汇总.音频id（按音频角色/剧情角色匹配角色名） | 火山引擎语音 ID |
| filename_prefix | Excel：音频汇总.音频id | 输出文件名前缀 |
| emotion | Excel：音频汇总.音频情感 或 配置 emotion_map | 情感参数 |
| config_path | API：config_path | 配置文件（appid、token 等） |

### 3.4 数据来源：Excel 表与列

**主表：音频汇总**

| 列名 | 用途 | 必填/说明 |
|------|------|-----------|
| 剧集id | 剧集过滤 | 必填 |
| 分镜号 | 分镜标识 | 必填 |
| 剧情角色 | 与音频角色二选一或一致，用于查找语音（见下） | 可选 |
| 音频角色 | 与剧情角色一起决定「角色名」，用于查角色汇总的 音频id/参考音色 | 可选 |
| 音频情感 | 火山引擎情感；ComfyUI 不使用 | 可选 |
| 音频id | 输出文件名前缀，且用于区分轨道 | 必填（无则跳过该轨） |
| 音频图片id | 备注/关联用 | 可选 |
| 音频内容 | 要合成的文本内容 | 必填（无则跳过） |
| 时长 | 备注用 | 可选 |
| 输出视频id | 备注用 | 可选 |
| 备注、参考样本 | 参考样本可作为 ComfyUI 的参考音频路径 | 可选 |

**关联表：角色汇总**

| 列名 | 用途 | 说明 |
|------|------|------|
| 角色名 | 与音频汇总的「音频角色/剧情角色」匹配 | 匹配后取该角色的 音频id 或 参考音色 |
| 音频id | 火山引擎：作为 voice_type（语音 ID） |  |
| 参考音色 | ComfyUI（数字人声）：作为参考音频文件路径（ref_audio_path） | 兼容列名「参考图」 |

**语音/参考音频解析逻辑简述**

- **火山引擎**：音频角色/剧情角色 → 角色汇总.角色名 → 取 角色汇总.音频id 作为 voice_type；若无则用角色名或 default。
- **ComfyUI（Qwen3-TTS 等）**：同上匹配角色 → 取 角色汇总.参考音色（或参考图）作为参考音频路径；若无则尝试 音频汇总.参考样本，再否则用角色名或报错。

**ComfyUI 音频生成：参考音频路径说明**

- 参考音频**不是目录**，而是**单个音频文件的路径**（.wav / .mp3 等），用于 Qwen3-TTS 语音克隆。
- 路径来源优先级：**角色汇总.参考音色**（或兼容列 参考图） > **音频汇总.参考样本** > 用角色名当作路径（易报错）。
- 路径可为**绝对路径**或**相对路径**；相对路径以 **UICS 进程当前工作目录**（通常为 UICS 根目录）为基准。文件必须存在于 UICS 所在机器上，服务端会将该文件上传到 ComfyUI。

### 3.5 输出文件命名

- 以 音频汇总.音频id 为文件名前缀，格式由生成器决定（如 .wav、.mp3 等）。

### 3.6 示例

**请求体示例（POST /api/generate/audio）**

```json
{
  "output_dir": "./output",
  "episode_filter": "EP01",
  "shot_filter": "EP01_SQ01",
  "generator_type": "volcengine",
  "config_path": null,
  "comfyui_server": "127.0.0.1:8188",
  "workflow_path": null
}
```

数字人声接口 **POST /api/generate/audio_digital** 请求体类似，但 `generator_type` 固定为 comfyui，无需传。

**音频汇总表示例**

| 剧集id | 分镜号   | 剧情角色 | 音频角色 | 音频id       | 音频内容                 | 音频情感 |
|--------|----------|----------|----------|--------------|--------------------------|----------|
| EP01   | EP01_SQ01 | 林婉     | 林婉     | EP01_SQ01_A01 | 这是什么？我没下载过啊。 | 自然     |
| EP01   | EP01_SQ01 | 李哲     | 李哲     | EP01_SQ01_A02 | 给你的，打开看看。       | 平静     |

- 每行一条「轨道」：同一分镜可有多条（多句台词），用 音频id 区分。生成时用「音频角色」或「剧情角色」去角色汇总里找 音频id（火山引擎）或 参考音色（ComfyUI）作为语音/参考音频。

**角色汇总表示例（与音频生成相关，含参考音色）**

| 角色名 | 音频id    | 参考音色 |
|--------|-----------|----------|
| 林婉   | S_linwan  | voice_ref/linwan.wav |
| 李哲   | S_lizhe   | /data/ref_audio/lizhe.mp3 |

- **参考音色**列：ComfyUI 数字人声时使用，填该角色克隆用的**参考音频文件路径**。可为相对路径（相对 UICS 根目录，如 `voice_ref/linwan.wav`）或绝对路径（如 `/data/ref_audio/lizhe.mp3`）。服务端会按该路径读文件并上传到 ComfyUI。
- **火山引擎**：音频汇总中「音频角色=林婉」→ 角色汇总.角色名=林婉 → 取 音频id=`S_linwan` 作为 voice_type。
- **ComfyUI（数字人声）**：同上匹配 → 取 参考音色 作为 ref_audio_path，音频内容作为 target_text，音频id 作为 filename_prefix；输出如 `EP01_SQ01_A01.wav`。

**音频汇总表示例（参考样本：轨道级参考音频）**

当某条轨道不用角色汇总的参考音色、而用本轨道自己的参考音频时，可填 **参考样本** 列（ComfyUI 时作为 ref_audio_path）：

| 剧集id | 分镜号   | 音频角色 | 音频id       | 音频内容     | 参考样本          |
|--------|----------|----------|--------------|--------------|-------------------|
| EP01   | EP01_SQ01 | 林婉     | EP01_SQ01_A01 | 这是什么？   | voice_ref/linwan.wav |
| EP01   | EP01_SQ02 | 路人甲   | EP01_SQ02_A01 | 您好。       | ref/guest_01.wav   |

- 参考样本为**可选**；不填时仍按「音频角色 → 角色汇总.参考音色」查找。填了则 ComfyUI 优先用该轨的参考样本路径。

**输出示例**

- 生成成功后，输出目录下会出现以 音频id 为前缀的文件，如 `EP01_SQ01_A01.wav`、`EP01_SQ01_A02.wav`。
